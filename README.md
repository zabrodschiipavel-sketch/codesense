# CodeSense

Тренажёр насмотренности: получаешь фрагмент кода на случайном языке, объясняешь словами, что он
делает, и получаешь оценку понимания в процентах плюс эталонный разбор.

Идея в том, чтобы читать, а не писать. Языки выпадают случайно из ~110 штук — от Python до APL,
Agda и COBOL. Знать язык не требуется и не предполагается: задача — вычитать смысл из незнакомого
синтаксиса.

## Запуск

```
run.cmd
```

Откроется http://localhost:8321. Нужен `DEEPSEEK_API_KEY` в переменных окружения
(`setx DEEPSEEK_API_KEY "<ключ>"`, потом заново открыть терминал).

Первый раз поставь зависимости: `pip install -r requirements.txt`

## Как устроен раунд

1. **Новый код** — сервер выбирает случайные сложность, язык и тему, DeepSeek генерирует фрагмент
   без единого комментария (комментарий = готовый ответ) вместе с тремя подсказками и эталонным
   разбором. Разбор лежит на сервере и до ответа клиенту не уходит.

   Язык и тему выбирает именно сервер, а не модель. Отданные на откуп модели, они схлопываются:
   язык сползает в Python, тема — в подсчёт слов, причём просьба «возьми другую тему» просто
   игнорируется. Явно названные язык и тема эту колею ломают.
2. **Подсказки** открываются по очереди и раскрывают смысл постепенно: синтаксис → контекст →
   почти прямая наводка. Каждая стоит −10% от очков. Счётчик живёт на сервере, а не в браузере.
3. **Проверка** — твой текст уходит в DeepSeek вместе с кодом и эталоном. Оценивается только
   понимание: формулировки, термины и грамматика не штрафуются.

## Карточки и интервальное повторение

Второй режим — карточки, как при изучении языка, только вместо слов конструкции: `|>`, `?.`,
`<<>>`, `cong`. На лице конструкция или мини-фрагмент, на обороте объяснение.

Карточки заводятся **сами из твоих ошибок**. Оценщик, уже разобравший код и твой ответ, тем же
вызовом возвращает 1–3 карточки по тому, что ты упустил, — лишнего запроса к API это не стоит.
Провалил Kotlin с `graph[node]?.any { ... } ?: false` — завтра эта конструкция спросит тебя сама.
Для ширины есть кнопка «Собрать колоду»: выбираешь язык, DeepSeek выдаёт 10–30 карточек по его
характерным местам.

Проверка — выбор из четырёх вариантов (`1`–`4` с клавиатуры, `Enter` — дальше). Модель тут не
участвует: она уже оценила тебя, когда карточка рождалась, а прогон из 30 карточек не должен
превращаться в полчаса ожидания.

**Дистракторы берутся из оборотов других карточек того же языка**, а не выдумываются моделью:
это бесплатно, мгновенно, работает без сети и правдоподобно по построению — каждый неверный
вариант описывает настоящую конструкцию этого же языка. Для `|>` в Elixir вариантами встанут
`receive`, `case` и pin `^`, и выбор превращается из узнавания в различение — то есть ровно в
тот навык, ради которого всё затевалось.

Отсюда следствие: **язык становится честным примерно от 4 карточек**. Если их меньше,
дистракторы подтягиваются из других языков и отсекаются с одного взгляда. Собери колоду по
такому языку — и варианты станут осмысленными.

Оговорка про метод: выбор из вариантов слабее свободного припоминания — узнать правильный
ответ легче, чем воспроизвести его, и это даёт некоторую иллюзию знания. Близкие однотипные
дистракторы этот эффект гасят, но не устраняют полностью.

В разделе два независимых контрола, и их легко перепутать, поэтому они подписаны:
**«Повторять»** слева — фильтр очереди (все языки вперемешку или один конкретный, с числом
карточек к повторению), **«Пополнить»** справа — язык и размер новой колоды. Плашка в шапке
всегда показывает общий долг по всем языкам, а строка прогресса — текущую отфильтрованную
сессию.

Алгоритм — SM-2 с дельтами лёгкости из Anki (мягче классических: провал роняет лёгкость на
0.20, а не на 0.80). Интервалы при ровных верных ответах: 1 → 6 → 15 → 38 → 95 дней. Промах
возвращает карточку в конец текущей сессии.

Выбор из вариантов даёт только «попал / не попал», а этого мало: если сводить всё к
`again`/`good`, лёгкость перестаёт двигаться (у `good` дельта нулевая) и SM-2 вырождается в
Лейтнера. Поэтому вторым сигналом берётся **время на ответ** — доступный здесь прокси беглости:

| Ответ | Оценка SM-2 |
| --- | --- |
| мимо | `again` |
| верно, до 6 с | `easy` |
| верно, до 20 с | `good` |
| верно, дольше | `hard` |

Пороги — `FAST_SECONDS` и `SLOW_SECONDS` в `srs.py`.

## Очки

```
очки = процент × (1 − 0.1 × подсказки) × множитель сложности
```

Множители: просто ×1.0, средне ×1.3, сложно ×1.6. Три подсказки срезают 30%.
Раунд от 70% продлевает серию. Ранги: Новичок → Читатель (400) → Толмач (1200) →
Полиглот (2800) → Дешифровщик (5500) → Оракул (10000).

## Файлы

| Файл | Что внутри |
| --- | --- |
| `app.py` | HTTP-эндпоинты, подсчёт очков |
| `deepseek.py` | Клиент DeepSeek: промпты генерации, оценки и колод |
| `languages.py` | Пулы языков, их развесовка по сложности, список тем |
| `srs.py` | Интервальное повторение: SM-2, интервалы, сроки |
| `quiz.py` | Сборка четырёх вариантов ответа и подбор дистракторов |
| `storage.py` | Банк сниппетов, карточки, прогресс, ранги, статистика |
| `static/index.html` | Весь интерфейс одной страницей |
| `data/snippets.json` | Все сгенерированные фрагменты с подсказками и разборами |
| `data/progress.json` | История раундов и очки |
| `data/cards.json` | Карточки и их состояние в SM-2 |

`data/` — обычный JSON, читается глазами и правится руками. Ничего не теряется: сгенерированный
фрагмент сохраняется до того, как ты на него ответишь, и если DeepSeek недоступен, раунд берётся
из банка среди непройденных.

## Настройки

Всё работает на одной модели `deepseek-v4-flash`. Она размышляет по умолчанию, а
`reasoning_effort` регулирует глубину (`low|medium|high|max|xhigh`); значение `off` отключает
размышления совсем.

```
CODESENSE_MODEL               (по умолчанию deepseek-v4-flash)
CODESENSE_REASONING_GENERATE  (по умолчанию off)
CODESENSE_REASONING_GRADE     (по умолчанию high)
```

Генерация идёт без размышлений: замерено на этом промпте — 6–13 секунд против 30–70 с ними, при
неотличимом качестве кода. Писать идиоматичный фрагмент модель умеет и так, думать там не над чем.

Оценке, наоборот, поставлен `high`: судить свободный текст против кода — самое трудное здесь.
На проверочном прогоне flash с `high` дал 95% там, где pro давал 88%, и при этом сохранил
нужный порядок — уверенно неверный ответ 5%, ленивый 15%. То есть flash чуть добрее на верхнем
краю шкалы и реже цепляется к тонкостям, но не ломается.

## Известные ограничения

- Подсветка синтаксиса — highlight.js с CDN, догружает грамматику под язык на лету. Для языков
  без своей грамматики (Agda, Gleam, Move) берётся ближайшая родственная, для APL/J/K/BQN
  подсветка отключена. Без интернета до CDN код показывается моношрифтом — на суть не влияет.
- Незавершённый раунд живёт в памяти процесса: перезапуск сервера в середине раунда обнулит
  счётчик подсказок, надо взять новый фрагмент. Сам фрагмент при этом остаётся в банке.
- Приложение однопользовательское и без авторизации — рассчитано на localhost.
